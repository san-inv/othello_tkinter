# コントリビューションガイド

このプロジェクトへの貢献に興味を持っていただきありがとうございます。

## 貢献の方法

### バグ報告

バグを発見した場合は、以下の情報を含めて Issue を作成してください：

1. バグの概要
2. 再現手順
3. 期待される動作
4. 実際の動作
5. 環境情報（OS、Pythonバージョン）

### 機能リクエスト

新機能のアイデアがある場合は、Issue を作成して提案してください。

### プルリクエスト

1. このリポジトリをフォークします
2. 新しいブランチを作成します
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. 変更を加えます
4. コードスタイルを確認します
   ```bash
   # flake8がインストールされている場合
   flake8 othello.py
   ```
5. 変更をコミットします
   ```bash
   git commit -m "Add: 機能の説明"
   ```
6. ブランチをプッシュします
   ```bash
   git push origin feature/your-feature-name
   ```
7. プルリクエストを作成します

## コーディング規約

- PEP 8 に準拠してください
- 型ヒントを使用してください
- 関数やクラスにはdocstringを記述してください
- 変数名・関数名は意味のある名前を使用してください

## コミットメッセージ

以下のプレフィックスを使用してください：

- `Add:` 新機能の追加
- `Fix:` バグ修正
- `Update:` 機能の更新・改善
- `Remove:` 機能・コードの削除
- `Refactor:` リファクタリング
- `Docs:` ドキュメントの変更
- `Style:` コードスタイルの変更（動作に影響しない）

## 質問

質問がある場合は、Issue を作成してください。
